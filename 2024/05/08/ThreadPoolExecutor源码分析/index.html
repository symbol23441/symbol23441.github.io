2、源码剖析

2.1 核心属性ctl

​ ctl通过AtomicInteger来保证操作的原子性，32为的整形，高3为表线程池状态，第29为表工作线程的数量。因此，通过ctl便可计算得线程池的状态与工作线程数量。

线程池状态（5种）：

state value 二进制 说明
RUNNING -1 111 执行处理任务，并阻塞队列中的任务
SHUTDOWN 0 000 不接受新的任务，正在处理、队列中的任务会继续执行
STOP 1 001 不接受新的任务，中断正在处理线程，不执行队列剩余任务
TIDYING 2 010 过渡阶段，代表线程池马上关闭
TERMINATED 3 011 由TIDYING经过terminated方法转变

2.2 execute()

3个核心分支：

  1. 核心线程未满，创建核心线程执行。（懒加载）
  2. 核心线程已满，阻塞队列未满，加队列。
  3. 核心线程满、队列满，加非核心线程
  4. 线程池状态非运行、全满，拒绝策略
