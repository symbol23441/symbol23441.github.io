一、什么是 JMM（Java Memory Model）？

JMM，全称 Java Memory Model（Java 内存模型），是 Java 语言对多线程环境中 共享变量的读写操作的行为规范。

它定义了：

  • 多线程之间 共享变量 如何被读/写。
  • 编译器、JVM、硬件如何协同保证 Java 程序的并发安全。

JMM 不是一块物理内存，而是一个 抽象的规范，解决的是“多线程之间可见性、有序性、原子性”的问题。

JMM 是 Java 规定线程读写共享变量的规则，定义了内存可见性、原子性、有序性，保证多线程程序能安全、稳定运行。

二、JMM 的核心问题

1. 可见性

  • 一个线程对变量的修改，另一个线程是否能立即看到？
  • 比如一个线程修改了共享变量 flag = true，另一个线程什么时候能看到？

2. 原子性

  • 一个操作是不可分割的，比如 i++ 并不是原子的，它其实包含：

    • 读变量
    • 自增
    • 写变量

    可用synchronized来保证方法代码段的原子性

3. 有序性

  • 指令是否按照代码的顺序执行？
  • JMM 允许编译器和 CPU 进行指令重排序优化，只要最终结果与单线程一致即可（as-if-serial）。

三、JMM 的内存模型结构

JMM 抽象出两部分内存：

含义：

  • 主内存（Main Memory）：存放所有共享变量
  • 工作内存（每个线程独有）：存储主内存中共享变量的副本
  • 线程对变量的操作，必须在自己的工作内存中进行
  • 不允许一个线程直接访问另一个线程的工作内存

四、JMM 定义的 8 种内存交互操作

为了让线程之间正确读写变量，JMM 定义了 8 个原子性指令：

操作 含义
lock 给变量加锁，独占
unlock 解锁，释放控制权
read 从主内存读取变量值
load 把 read 得到的值放入工作内存副本
use 把工作内存变量用于计算
assign 给工作内存变量赋值
store 把工作内存变量写回主内存
write 将 store 的值写入主内存变量

📌 读操作 = read + load，写操作 = assign + store + write

五、JMM 与关键字的关系

关键字 与 JMM 的关系
volatile 保证 可见性、禁止重排序
synchronized 保证 可见性 + 原子性 + 有序性（通过 lock/unlock）
final 保证 构造过程中的可见性（不可变对象）

对 volatile 变量：

操作 行为
写（赋值） 线程 直接将值写入主内存（store + write），并 刷新工作内存中的副本
读（取值） 线程 从主内存中强制读取最新值（read + load），不使用工作内存中的旧副本

所以，volatile 保证了可见性，但 不保证原子性！

volatile 写操作的完整交互流程

假设线程 A 执行：

🌊 内存交互流程如下（线程 A）：

  1. assign：线程 A 在工作内存中把 flag = true
  2. store：把这个值准备写回主内存（进入写缓冲区或写通道）
  3. write：将值真正写入主内存中 flag 的位置
  4. 🚫 清除其他线程工作内存中对 flag 的缓存副本（实现强制刷新）

✅ 重点：volatile 写操作 直接写主内存，并强制使其他线程对该变量的工作内存副本失效

volatile 读操作的完整交互流程

设线程 B 执行：

🌊 内存交互流程如下（线程 B）：

  1. read：线程 B 从主内存中读取 flag 的最新值
  2. load：将读取的值加载到自己工作内存中的 flag 副本
  3. use：使用这个值进行判断（if 语句）

✅ 重点：volatile 读操作 不使用工作内存的副本，一定从主内存中重新读取最新值

六、JMM 的 happens-before 规则（重要）

JMM 使用 happens-before 关系定义哪些操作必须对另一个线程“可见”：

happens-before 规则：

规则 描述
程序顺序规则 单线程内，代码按顺序执行
锁规则 解锁先于加锁（释放锁前的修改，对后面拿锁可见）
volatile 规则 写 volatile 先于读
线程启动规则 start() 先于子线程动作
线程终止规则 join() 先于线程结果可见
传递性规则 A → B，B → C，则 A → C

📌 只要两个操作之间存在 happens-before，就保证：

  • 前者的修改对后者是可见的
  • 前者的执行顺序在后者之前

七、JMM 解决的问题场景（举例）

  • 如果 running 没加 volatile，线程 B 可能永远看不到线程 A 改的 false（可见性问题）
  • 加了 volatile，JMM 保证线程 A 的写入操作 write 后会对 B 的 read 可见。

JMM 是 Java 规定线程读写共享变量的规则，定义了内存可见性、原子性、有序性，保证多程程序能安全、稳定运行。
