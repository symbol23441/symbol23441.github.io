内网穿透(by ssh or frp)

内网穿透是一种技术，用于通过公网访问位于防火墙或路由器（NAT）后的内网设备或服务。由于内网设备通常没有公网IP，外部无法直接访问。内网穿透突破这一限制，在保证安全的前提下实现外部对内网主机或服务的访问。

通常实现方式：反向代理、P2P打洞、静态端口映射。P2P对路由器NET类型有要求，端口映射一般需要网关路由的权限。因此，适用性最高的是反向代理的方式。

虽然市面也有一些内网穿透的服务，包括； natapp、coplar、花生壳等，如果你不想折腾，也可以直接使用。但折腾一下，往往会省钱！

本文提供两种方式的示例：SSH内网穿透、frp内网穿透

详细如下：

1、ssh 内网穿透（代理）

步骤：

  1. 内网本地主机：打开ssh服务
  2. 外网IP服务器：打开ssh服务、端口转发设置、防火墙or安全组开放端口

1.1 内网本地主机

补充：

基本指令 ssh user@host

-p 指定端口。默认22。

例：ssh -p 2222 user@host

-L <本地端口>:<目标地址>:<目标端口>

用途：将本地端口转发到远程主机所能访问的另一个主机上。

例： ssh -L 8080:localhost:80 user@remote 本地访问 localhost:8080 会转发到远程主机的 localhost:80

ssh -R <远程端口>:localhost:<本地服务端口> user@host

例：ssh -R 9000:localhost:8080 user@cloud-server 云服务器上监听 9000，流量转发给你本地的 8080 端口

-N 表示不在远程执行任何命令，只建立隧道连接。无后续链接窗口

此外，保持连接不中断（需搭配 autossh）。autossh -M 0 -N -R 9000:localhost:22 user@host,autossh 会自动重连 SSH 隧道，适合长期运行场景。
