消息摘要算法

摘要：

  • 消息摘要算法是密码学中的基础模块，用于生成安全、唯一的数据摘要。
  • 它基于复杂的分块计算和状态压缩，确保数据的完整性验证。
  • SHA-256 是当前主流且安全的选项；MD5、SHA-1 已不再安全。
  • 摘要算法实际就是一种高安全级别的哈希算法。

1. 什么是消息摘要算法？

消息摘要算法（Message Digest Algorithm）是一类将任意长度输入压缩为固定长度摘要的函数。它广泛用于验证数据完整性、数字签名、密码存储、区块链等安全场景。

其目标是生成一个唯一的“数据指纹”，具有如下特点：

  • 输出长度固定（例如 SHA-256 输出 256 位）
  • 原文不可逆推出（单向性）
  • 两个不同输入几乎不可能得出相同输出（抗碰撞）
  • 输入微小变化会造成输出完全不同（雪崩效应）

2. 消息摘要 ≈ 哈希算法？

消息摘要算法本质上是密码学意义上的哈希函数，但比普通哈希函数具有更严格的安全标准。

哈希函数的基本特性：

  1. 确定性：相同输入一定产生相同输出。
  2. 输出固定长度：不论输入多长，输出长度相同。
  3. 快速计算：能够高效计算出结果。
  4. 抗碰撞：几乎不可能找到两个不同输入对应相同输出。
  5. 抗原像攻击：无法从摘要反推出原始输入。
  6. 抗二次原像攻击：无法从一个输入找到另一个具有相同摘要的输入。

关系说明：

项目 哈希函数（广义） 消息摘要算法（密码学）
用途 哈希表、负载均衡、分布式索引等 数字签名、校验、身份验证等
安全要求 低 高
示例 Java hashCode、MurmurHash SHA-256、SHA-3、BLAKE3

消息摘要算法是哈希函数的一个“安全增强版本”。

3. 摘要算法工作原理（以 SHA-256 为例）

流程概述：

  1. 填充（Padding）
    将原始数据填充为 512 位的整数倍。规则是：
    • 末尾添加 1
    • 添加足够多的 0
    • 追加原始长度的二进制表示
  2. 分块处理
    每块大小为 512 位，逐块进行计算。
  3. 初始化状态（IV）
    使用一组固定的初始值作为起点，例如 8 个 32 位常量。
  4. 主循环（压缩函数）
    每一块数据将与当前状态混合，经过位运算（如与、或、异或、移位）、常数操作等计算出新状态。
  5. 输出摘要
    所有块处理完毕后，拼接状态变量形成最终的摘要输出（如 SHA-256 得到 256 位摘要）。

4. 常用消息摘要算法对比

算法 输出长度 状态 应用场景
SHA-256 256 bit 推荐 Web 安全、数字签名、区块链
SHA-512 512 bit 推荐 高安全场景，如身份认证
SHA-3 可变长度 推荐 替代 SHA-2，设计更安全
BLAKE3 256 bit 新兴算法 快速、并行、移动端友好
MD5 128 bit 不安全 只用于非安全场景（如文件去重）
SHA-1 160 bit 不安全 已遭碰撞攻击，建议停用

6. 安全性从何而来？

安全特性 设计依据
抗碰撞 多轮压缩函数设计与状态扰动机制
雪崩效应 非线性布尔函数与状态混合策略
不可逆 不保留中间状态，无法逆推出原文
恒定长度输出 状态变量组合映射输出

摘要算法的核心安全性来自其“无法逆推 + 无法伪造 + 不易冲突”的特性。
